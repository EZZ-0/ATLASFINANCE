                     3. HOW TO START A SESSION
================================================================================
================================================================================

OPTION A: FULL SESSION (Recommended)
────────────────────────────────────

  1. Open TWO Cursor windows/chats

  2. In Window 1 (Architect):
     - Paste the ARCHITECT PROMPT from Section 6
     - Add your request: "I want to build [feature]"
     - Architect will read files, update ROADMAP.md, start working

  3. In Window 2 (Executor):
     - Paste the EXECUTOR PROMPT from Section 6
     - Or just send: .
     - Executor enters continuous mode, waits for tasks

  4. Let them work:
     - Architect creates tasks in INBOX_EXECUTOR.md
     - Architect posts [TASK_READY] in LIVE_CHAT.md
     - Executor sees signal, executes tasks
     - Both work continuously until done

  5. Session ends when Architect posts:
     [SESSION_COMPLETE]


OPTION B: ARCHITECT ONLY (Simple tasks)
───────────────────────────────────────

  For simple work that doesn't need delegation:
  - Just talk to Architect
  - They'll do everything themselves
  - Skip Executor entirely


OPTION C: CONTINUING A SESSION
──────────────────────────────

  If you need to restart after a break:
  - Check LIVE_CHAT.md for last state
  - Check ROADMAP.md for progress
  - Tell Architect: "Continue from where we left off"
  - Send "." to Executor to reactivate

                    4. HOW TO MONITOR PROGRESS
================================================================================
================================================================================

QUICK STATUS CHECK:
──────────────────

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  File to Check          │  What You'll See                             │
  ├─────────────────────────────────────────────────────────────────────────┤
  │  MISSION_CONTROL.md     │  Quick status table, priorities              │
  │  ROADMAP.md             │  Milestones, current objective, progress     │
  │  LIVE_CHAT.md           │  Real-time agent conversation                │
  │  ACTIVE_TASKS.md        │  What's currently being worked on            │
  │  COMPLETED_TASKS.md     │  What's finished                             │
  └─────────────────────────────────────────────────────────────────────────┘


WHAT TO LOOK FOR IN LIVE_CHAT.md:
─────────────────────────────────

  GOOD SIGNS (Session progressing):
    [SYNC] ............... Agents are online
    [TASK_READY] ......... Tasks being created
    [UPDATE] ............. Work is happening
    [DONE] ............... Tasks completing
    [VALIDATED] .......... Work is verified
    [SESSION_COMPLETE] ... All done!

  NEEDS ATTENTION:
    [WAITING_FOR_TASKS] .. Executor waiting (normal if Architect planning)
    [BLOCKED] ............ Something is stuck
    [DECISION_NEEDED] .... You need to provide input
    [AGENT_ERROR] ........ Problem occurred - check immediately
    [PROTOCOL_BREAK] ..... Agent malfunction - check immediately


SESSION PROGRESS INDICATORS:
────────────────────────────

  Early Session:
    - Architect: [SYNC], updating ROADMAP.md
    - Executor: [WAITING_FOR_TASKS]
    - Normal - Architect is planning

  Mid Session:
    - [TASK_READY] appearing
    - [DONE] messages
    - [VALIDATED] confirmations
    - Good progress!

  Late Session:
    - [ALL_TASKS_ASSIGNED]
    - Final [DONE] messages
    - Approaching completion

  Session End:
    - [SESSION_COMPLETE]
    - Architect reports summary to you

                      5. WHEN TO INTERVENE
================================================================================
================================================================================

YOU MUST RESPOND WHEN:
──────────────────────

  ╔═══════════════════════════════════════════════════════════════════════════╗
  ║  [DECISION_NEEDED]                                                        ║
  ║  ──────────────────                                                       ║
  ║  Architect needs your input on something                                  ║
  ║  Action: Read their question, provide your decision                       ║
  ╚═══════════════════════════════════════════════════════════════════════════╝

  ╔═══════════════════════════════════════════════════════════════════════════╗
  ║  [AGENT_ERROR] @USER                                                      ║
  ║  ─────────────────────                                                    ║
  ║  An agent has a problem with their function                               ║
  ║  Action: Read the error, provide guidance or fix                          ║
  ║  The agent has STOPPED and is waiting for you                             ║
  ╚═══════════════════════════════════════════════════════════════════════════╝

  ╔═══════════════════════════════════════════════════════════════════════════╗
  ║  [PROTOCOL_BREAK] @USER                                                   ║
  ║  ───────────────────────                                                  ║
  ║  One agent reports the other isn't following protocol                     ║
  ║  Action: Check what's wrong, restart or guide the failing agent           ║
  ╚═══════════════════════════════════════════════════════════════════════════╝

  ╔═══════════════════════════════════════════════════════════════════════════╗
  ║  [BLOCKED]                                                                ║
  ║  ──────────                                                               ║
  ║  Something external is blocking progress                                  ║
  ║  Action: Check BLOCKERS.md, help resolve                                  ║
  ╚═══════════════════════════════════════════════════════════════════════════╝


OPTIONAL INTERVENTIONS:
───────────────────────

  • Change priorities mid-session
    → Tell Architect: "Stop current work, focus on X instead"

  • Add new requirements
    → Tell Architect: "Also add Y to the scope"

  • Check on quiet session
    → Ask Architect for [STATUS]

  • Restart stuck Executor
    → Send "." again

  • Write directly to LIVE_CHAT.md
    → Format: [TIMESTAMP] [USER]: @ARCHITECT or @EXECUTOR [message]
    → Agents will see it on their next check


NO INTERVENTION NEEDED FOR:
───────────────────────────

  • [WAITING_FOR_TASKS] - Normal, Executor is standing by
  • [WAITING] - Normal, task being written
  • [Q] and [A] - Agents talking to each other
  • [LOCK] and [UNLOCK] - File coordination
  • [UPDATE] - Progress updates

                       7. TROUBLESHOOTING
================================================================================
================================================================================

PROBLEM: Executor not responding
────────────────────────────────
  Symptoms: No messages from Executor in LIVE_CHAT
  Solution: Send "." again to trigger them
  If still nothing: Check if their session is still open


PROBLEM: Architect not creating tasks
─────────────────────────────────────
  Symptoms: Executor shows [WAITING_FOR_TASKS] for too long
  Solution: Ask Architect directly: "What's the status? Create tasks for Executor."


PROBLEM: Session seems stuck
────────────────────────────
  Symptoms: No new messages in LIVE_CHAT for a while
  Solution: 
    1. Ask Architect: "[STATUS] request"
    2. Check ACTIVE_TASKS.md for what's being worked on
    3. Check BLOCKERS.md for issues


PROBLEM: [AGENT_ERROR] reported
───────────────────────────────
  Symptoms: Agent posts error and stops
  Solution:
    1. Read the error details carefully
    2. Address the issue they describe
    3. Tell them to continue: "Issue resolved, continue working"


PROBLEM: [PROTOCOL_BREAK] reported
──────────────────────────────────
  Symptoms: Agent reports other agent not following protocol
  Solution:
    1. Check what the violation is
    2. Talk to the offending agent
    3. Remind them of the protocol
    4. If needed, restart their session with the prompt


PROBLEM: File conflicts
───────────────────────
  Symptoms: Both agents trying to edit same file
  Solution:
    1. Check LOCKED_FILES.md
    2. Tell one agent to wait
    3. Ensure they post [LOCK] before editing


PROBLEM: Need to stop mid-session
─────────────────────────────────
  Symptoms: You need to pause or cancel
  Solution:
    1. Tell Architect: "Pause session, save state"
    2. Architect updates ROADMAP.md with current state
    3. Both agents stop
    4. Resume later with "Continue from where we left off"
