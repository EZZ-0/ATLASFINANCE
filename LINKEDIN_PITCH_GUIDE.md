## TALKING POINTS (For Q&A)

### "Why did you build this?"
"Two reasons: I wanted to deeply understand how professional investors analyze companies, and I saw a gap - institutional tools cost $20k+/year, free tools are basic, nothing in between. This fills that gap for students and retail investors."

### "What was the hardest part?"
"Understanding the finance. The code is straightforward - DCF is just math. But knowing WHEN to use certain assumptions, HOW to interpret ratios, WHAT makes a red flag actually concerning - that required reading 50+ investment memos and finance textbooks."

### "Is the data accurate?"
"Transparent answer: 90-95% accurate for large-cap stocks, less reliable for small-caps. I validate against multiple sources and show disclaimers. It's directionally correct - good for learning and initial screening, not for making million-dollar decisions."

### "What would you improve?"
"Three things: 1) Performance - add caching to speed up data extraction. 2) Mobile - currently desktop only. 3) Data coverage - integrate paid data sources like Polygon or FactSet for production use."

### "Can I use this for real investing?"
"Educational purposes only. Always verify with professional sources and consult a financial advisor. The tool shows you HOW to analyze, not WHAT to decide."

### "How is this different from free tools?"
"Most free tools show basic metrics (P/E, revenue). This performs actual analysis - building a DCF model, generating investment theses, detecting red flags. It's the difference between a calculator and a financial analyst."

### "Did you use AI to build this?"
"Yes - I used AI-assisted development (Claude/Cursor) for about 70% of implementation. I focused my effort on: financial modeling logic, architecture decisions, and validation. This reflects modern development practices - AI accelerates coding, but domain expertise (finance knowledge) is still critical."
